# SchemaFlow Roadmap

## Current Status

**Version**: 0.1.0 (MVP in development)

## Milestones

### MVP (0.1.0) - In Progress

**Goal**: Core schema lifecycle management

**Features**:
- ✅ Schema Registry (CRUD)
- ✅ Version Management
- ✅ Schema Diff
- ✅ Release Management
- ✅ Rollback

**Excluded**:
- ❌ Designer (visual editor)
- ❌ AI features
- ❌ Rule Engine
- ❌ Authentication

**Target**: Q2 2024

---

### Alpha (0.2.0) - Planned

**Goal**: Enhanced usability and stability

**Features**:
- [ ] Import/Export (JSON, YAML)
- [ ] Schema templates library
- [ ] API versioning
- [ ] Webhooks
- [ ] Basic authentication (API keys)
- [ ] PostgreSQL support
- [ ] Docker deployment

**Target**: Q3 2024

---

### Beta (0.3.0) - Planned

**Goal**: Enterprise readiness

**Features**:
- [ ] Multi-tenancy
- [ ] Teams and permissions
- [ ] Audit logging
- [ ] Schema validation on save
- [ ] Performance optimizations
- [ ] CLI tool
- [ ] SDKs (JavaScript, Python)

**Target**: Q4 2024

---

### v1.0.0 - Planned

**Goal**: Production ready

**Features**:
- [ ] Full authentication (SSO, OAuth)
- [ ] Advanced search
- [ ] Analytics dashboard
- [ ] SLA guarantees
- [ ] Enterprise support

**Target**: Q1 2025

## Future Ideas

### Designer (Post v1.0)

Visual form builder for non-technical users.

- Drag-and-drop interface
- Live preview
- Field type library
- Conditional logic (simple)

**Status**: Out of scope until v1.0

### AI Features (Post v1.0)

AI-assisted schema development.

- Generate schema from description
- Suggest improvements
- Auto-generate test data
- Detect breaking changes

**Status**: Out of scope until v1.0

### Rule Engine (Post v1.0)

Custom validation and transformation rules.

- Expression-based validation
- Field dependencies
- Dynamic visibility
- Custom functions

**Status**: Out of scope until v1.0

## Feature Requests

Want to request a feature?

1. Check [existing issues](https://github.com/schemaflow/schemaflow/issues)
2. Open new issue with:
   - Clear description
   - Use case
   - Proposed solution (optional)

We'll tag it and discuss in roadmap planning.

## Contributing to Roadmap

See something you'd like to work on?

- Comment on issue to claim
- Create spec following [Spec-Driven Development](docs/guides/03-spec-driven-development.md)
- Submit PR

Priorities can shift based on:
- Community feedback
- Contributor interest
- Technical constraints
